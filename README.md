# AnyQuote

一个面向 **Scripting App / iOS 小组件场景** 的「每日一言」项目。

AnyQuote 默认接入一言接口，同时支持接入多个自定义内容源。它的目标不是做一个只能看默认文案的小组件，而是让你可以把桌面上的一句话，替换成你自己的摘抄、语录、文案库，甚至是任意兼容结构的外部 API。

## 项目定位

- 默认可用：不配置也能直接显示内容
- 高度开放：支持切换多个自定义来源
- 内容优先：围绕“一句话 + 来源”做简洁展示
- 轻量设置：在详情页内直接管理刷新和数据来源

## 当前功能

- 默认一言来源
- 多个自定义来源管理
- 当前来源切换
- 恢复默认来源
- 自动刷新开关
- 刷新间隔设置
- 手动立即刷新
- 兼容一言字段结构的外部 API 解析

## 当前限制

当前版本的外部来源规则比较克制，目的是先把可用性做稳：

- 只支持 `GET`
- 只支持 `JSON`
- 暂不支持 `API Key`
- 暂不支持自定义请求头
- 暂不支持 POST Body

如果你的接口需要鉴权，当前版本还不能直接接入。

## 自定义 API 兼容规则

AnyQuote 以一言原始返回结构作为默认协议。

### 必填字段

以下字段至少存在一个：

- `hitokoto`
- `content`

### 可选字段

- `from`
- `from_who`
- `type`
- `length`

### 示例 1：一言原始结构

```json
{
  "hitokoto": "我们终将上岸，阳光万里。",
  "from": "摘录",
  "from_who": "你自己",
  "type": "a",
  "length": 13
}
```

### 示例 2：自定义兼容结构

```json
{
  "content": "今天也要认真生活。",
  "from": "便签",
  "from_who": "自己",
  "type": "日常",
  "length": 9
}
```

## 使用方式

### 1. 默认使用

导入脚本后直接运行即可，AnyQuote 会默认使用一言来源。

### 2. 添加自定义来源

1. 打开详情页
2. 点击右上角“设置”
3. 进入“数据来源”
4. 新增来源并填写：
   - 名称
   - URL
5. 保存后会立即切换到这个来源并刷新内容

### 3. 切换或恢复默认来源

- 可以在多个自定义来源之间切换
- 可以随时恢复为默认一言来源

## 核心设计思路

### 1. 一言结构作为统一兼容协议

AnyQuote 不要求所有外部 API 都长得一模一样，但要求它们至少提供可映射到一言结构的字段。这样可以把接入门槛降到很低，同时保持内部渲染逻辑统一。

### 2. 默认来源与自定义来源并存

默认来源始终可用，自定义来源独立管理。这样既能保证开箱即用，也能满足进阶用户的个性化需求。

### 3. 切换来源时立即刷新

来源切换或编辑当前来源后，会立即清理旧缓存并请求新内容，避免不同来源的数据混在一起。

## 适合的使用场景

- 个人摘抄 / 语录展示
- 自建内容接口的小组件入口
- 团队文案池、公告池、短句池展示
- 作为练手的 Scripting / Widget 项目

## 常见问题

### 为什么保存来源后提示刷新失败？

通常是下面几种情况：

- URL 不正确
- 接口没有返回合法 JSON
- JSON 中没有 `hitokoto` 或 `content`
- 接口需要鉴权

### 为什么当前版本不支持 API Key？

当前仓库先优先落地“默认来源 + 多来源管理 + 统一解析协议”这条主链路，鉴权支持会让设置结构和请求配置复杂很多，所以这版先没有接进去。

### 切换来源后为什么还保留旧内容？

这是刻意设计的兜底行为。如果新来源请求失败，界面会保留原来的内容，避免组件直接空掉。
